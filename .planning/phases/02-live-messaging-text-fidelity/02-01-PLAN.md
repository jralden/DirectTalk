## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | POST /messages text field | mitigate | appendMessage strips ANSI (no terminal-control injection into transcript); text stored as JSON string, never executed; rendering as text/HTML-safety is Phase 3 |
| T-02-02 | Spoofing | role detection via remoteAddress | accept | trusted home LAN per spec; loopback => host is sufficient, no auth wanted (REQUIREMENTS out-of-scope) |
| T-02-03 | DoS | SSE subscriber registry | mitigate | res.on('close') removes subscriber and clears heartbeat interval; empty Sets deleted to bound memory; broadcast writes wrapped in try/catch |
| T-02-04 | DoS | POST body size | mitigate | existing readBody 1MB cap (req.destroy) applies to /messages |
| T-02-05 | Tampering | :id path param | mitigate | regex `^/api/sessions/([a-z0-9-]+)/(stream|messages)$` constrains id; readSession null-guard -> 404; sessionPath rejects out-of-charset ids (no path traversal) |
| T-02-06 | DoS | crash on bad input | mitigate | handler-level try/catch -> 500 (existing); JSON.parse guarded -> 400; process stays alive |
</threat_model>
